# Agent 通用启动基线

> 这是所有 Agent 档案页共用的启动与自检基线。各 Agent README 只保留本机差异与专属能力，不再重复整套骨架。

## 通用目标

所有 Agent 的 README 统一回答 4 件事：

1. 它是谁，负责什么。
2. 它从哪里读取本机配置与技能。
3. 它需要同步哪些档案页。
4. 它与 CortexOS 的协作入口是什么。

## 通用自检骨架

每个 Agent README 默认都遵循以下流程：

1. **物理扫描**：读取对应客户端配置，确认 MCP 与 skills 的真实安装状态。
2. **档案校准**：对比 `docs/agents/<agent>/` 下的 `manifest.md`、`mcp.md`、`skills.md`。
3. **状态同步**：如果物理环境变了，必须先更新档案，再继续执行任务。

## 通用档案室索引

- `manifest.md`：身份定位、专长、推荐工具链
- `mcp.md`：MCP 服务清单与连接状态
- `skills.md`：本机已安装技能与用途

## 通用维护约束

- 档案页是文档化镜像，不是唯一真实配置源。
- 物理配置始终以 `~/.gemini/`、`~/.claude/`、`~/.codex/` 为准。
- README 只写差异项，不再重复整套说明模板。
