# 复盘记录：从业务 SDK 到通用 Chatbot UI SDK 的转型

**日期**: 2026-02-06
**老爹满意度**: 已成功转型并同步到新仓库

## 1. 核心洞察 (Insights)
- **UI 是项目的灵魂**：用户对“逻辑通用”的需求绝不能以“牺牲 UI 细节”为代价。即便逻辑大改，原有的动画、阴影、RGB 边框必须一个像素不少地保留。
- **配置的“灵活性”**：不仅是支持 JSON，更要支持“半透明”配置——即开发者可以只传品牌名，其他文案自动匹配；也可以全量覆盖。

## 2. 自动化 SOP (今后执行此类任务的步骤)
1. **[审计]** 搜索所有品牌关键词，识别耦合点。
2. **[保护]** 将 `tsx` 文件中的 JSX 结构视为“不可变区域”。
3. **[剥离]** 将交互逻辑（State/Methods）整体搬迁至 `hooks/`。
4. **[模板化]** 将默认文案中的品牌词替换为动态变量。
5. **[注入]** 通过 `init({ ...config })` 实现配置热更新（通过销毁并重建根节点实现）。

## 3. 技术规约 (Rules Update)
- **CSS 前缀**: 必须使用统一的 Namespace (如 `cb-` 或 `pl-`)。
- **API 安全**: 演示页面禁止显示任何 API Key，必须在后台合并。
- **Logo 兼容性**: 同时支持 `logoUrl`, `logoSvg` 和原生 `ReactNode`。
