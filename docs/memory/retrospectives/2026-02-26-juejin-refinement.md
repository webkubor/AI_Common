# 复盘：2026-02-26 掘金发布人格解耦与工具链极致清理

## 1. 事件背景
在执行“llms.txt”深度技术文章发布任务时，发生了身份认知混乱，将“司南烛/微电影”的武侠人设带入了技术社区，并触发了冗余的脚注注入。同时，由于 `browser-use` 工具在复杂弹窗下的不稳定性，导致发布流程多次中断。

## 2. 核心问题诊断
- **身份粘连**：未能根据平台（掘金 vs 小红书）自动切换人格，导致技术文章显得不伦不类。
- **模板依赖**：过度依赖固定的“作者简介”模板，忽视了用户对纯净技术内容的需求。
- **工具链冗余**：保留了多个功能重叠且不稳定的浏览器控制扩展（browser-use, stitch），增加了系统的熵值。

## 3. 执行的改进措施（已实时生效）
- **人格逻辑隔离**：
    - **掘金** = 纯粹开发者（Zero humans, Zero faces, Zero private life）。
    - **小红书** = 司南烛（武侠叙事，生活化记录）。
- **工具链大扫除**：
    - 卸载并物理删除 `browser-use` 及其缓存。
    - 卸载连通性故障的 `Stitch`。
    - **确立 Chrome DevTools MCP 为唯一浏览器操作标准**。
- **发布规范固化**：
    - 强制执行“影子预览”：发布前检查正文尾部是否有冗余内容。
    - 禁用所有未经请求的自动化脚注。

## 4. 经验总结
> “不用的工具就是负债，混乱的人格就是噪音。”

在 AI 辅助开发的闭环中，**极简主义**不仅是审美，更是生产力。通过对工具链的物理清理和对人格的逻辑解耦，实现了发布效率与内容质量的双重提升。
