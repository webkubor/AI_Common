## 规则路由说明

外部大脑健康检查的完整清单与命令统一维护在：

- [外部大脑健康检查标准](../../ops/external-health-check.md)

本页只保留规则路由语义：

- 触发词仍然是 `自检 / 健康检查 / 系统检查 / health check`
- Agent 命中该规则时，应先读取运维版清单，再执行检查
- Sentinel 规则不再复制整份检查内容，避免双份维护

## Sentinel 额外约束

执行健康检查时，额外关注：

1. 是否存在路径、链接、结构层面的静态损坏
2. 是否存在凭证目录缺失或敏感配置漂移
3. 检查完成后是否把结果记录回日志或任务流
