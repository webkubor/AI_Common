- **执行优先级**: 执行任何实质性任务前，必须优先检索可用技能 ()。
- **工作流**: 必须遵循 '检索技能 -> 制定详细计划 -> 用户确认（如涉及重大变更） -> 执行' 的顺序，严禁直接跳过分析阶段进行代码写入。

## 📦 包发布前置校验 (NPM Publishing SOP)
- **唯一性检查**: 在创建任何新项目或首次执行发布前，**必须**先执行 `npm view <package-name>`。若返回结果非空，说明包名冲突，必须立即与用户协商改名。
- **作用域优先**: 优先建议用户使用作用域包 (Scoped Package, 如 @webkubor/xxx) 以避免公共命名空间的恶意占用或意外冲突。
